Les maquettes nous donnent une bonne vision du site définitif. Il ne reste donc plus qu'à convertir les maquettes en site web. Pour cela, nous allons utiliser deux langages de programmation : HTML et CSS. Le HTML va nous permettre de mettre en place les éléments de structure (image, texte, lien) et le CSS va s'occuper de tous les éléments de style. Les deux langages sont donc complémentaires.

HTML - Elements de structure

Une page de code HTML permet de lister tous les éléments que nous voulons retrouver sur le site :

  • Images
  • Textes
  • Titres
  • Liens
  • ...

Une page HTML est un langage de balisage. Chaque balise a une signification particulière. Il existe des référenciels de ces balises pour savoir comment les utiliser.

Si la syntaxe du HTML parait simple au premier abord, les pages peuvent rapidement devenir complexes. Il est donc primordial de formatter correctement vos fichiers HTML pour pouvoir les modifier facilement. Il est important également d'utiliser au maximum les balises sémentiques car elles permettent aux navigateurs et au robot google de comprendre la structure de la page.

CSS - Elements de style

Un fichier CSS contient les informations de style pour les éléments de structure mis en place dans les fichiers HTML:

  • Police
  • Couleur
  • Alignement
  • Dimension
  • ...

Ici également la syntaxe peut être comprise rapidement, mais les fichiers CSS mal bâtis peuvent devenir très pénibles à lire. Il convient donc dès le départ de penser à l'organisation du fichier et veiller à ne pas dupliquer les formats. C'est un investissement dans l'avenir. Plus le fichier est clair et lisible, plus le développeur va gagner du temps lors des évolutions et des corrections.
